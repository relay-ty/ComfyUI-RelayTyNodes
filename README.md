# ComfyUI-RelayTyNodes

ComfyUI 常用自定义节点库，提供开关、工具、Mask处理、字符串处理和数学计算等功能。

## 📦 安装

### 方式一：Git Clone

```bash
cd ComfyUI/custom_nodes
git clone https://github.com/relay-ty/ComfyUI-RelayTyNodes.git
```

### 方式二：手动下载

下载本仓库并解压到 `ComfyUI/custom_nodes/ComfyUI-RelayTyNodes` 目录。

### 依赖安装

```bash
pip install -r requirements.txt
```

**依赖说明**：
- `scipy`: 用于 Mask 处理和 BBox 转 Mask 的羽化功能
- `opencv-python`: 用于 Mask 描边功能
- `numpy`: 基础数值计算
- `color-matcher`: 用于颜色匹配算法
- `kornia`: 用于 GPU 加速的颜色匹配

## 📚 节点列表

本节点库包含 **22 个自定义节点**，分为以下几类：

---

### 🔌 开关节点

| 节点名称 | 说明 |
|----------|------|
| RT惰性开关 | 根据条件值选择并输出对应的输入，支持惰性求值 |
| RT双路惰性开关 | 双路版本的惰性开关，支持 True/False 两个分支 |

**特性**：
- 只有被选中的输入才会被计算
- 未被选中的输入可以为 None 而不影响执行
- 支持任意类型的输入输出

---

### 🛠️ 工具节点

| 节点名称 | 说明 |
|----------|------|
| RT分辨率选择器 | 快速选择标准分辨率值（360P、480P、720P、1080P 等） |
| RT比例选择器 | 选择常用宽高比（1:1、4:3、16:9、9:16 等） |
| RT比例检测器 | 根据输入图像自动检测并输出最接近的标准宽高比 |

**特性**：
- 所有工具节点都内置帮助文档
- 点击节点右上角的 ❓ 按钮可查看详细使用说明

---

### 🎭 Mask 处理节点

| 节点名称 | 说明 | 依赖 |
|----------|------|------|
| RT遮罩扩展/收缩 | 调整 Mask 区域大小，正值膨胀、负值腐蚀，支持 GPU 分块 | - |
| RT遮罩高斯羽化 | 对 Mask 进行高斯模糊羽化，平滑边缘过渡 | - |
| RT遮罩分离 | 分离并跟踪 Mask 区域，返回各个独立区域 | scipy |
| RT获取Track | 从打包的 tracks 中取出指定索引的 track | - |
| RT复制遮罩批次 | 复制遮罩批次，用于批量处理 | - |
| RT从批次获取遮罩 | 从批次中获取指定索引的遮罩 | - |
| RT图像通道转遮罩 | 从图像指定色彩通道提取 Mask，支持亮度/RGB/HSV通道 + 色阶调整 | - |
| RT遮罩转图像 | 将 Mask 转换为 RGB 图像 | - |
| RT遮罩描边 | 在图像上绘制 Mask 各独立区域的轮廓线 | OpenCV |
| RT遮罩绘制 | 在图像上绘制 Mask，支持多种颜色格式（RGB、RGBA、Hex） | - |
| RTBBox转遮罩 | 将边界框（BBox）列表转换为 Mask 图像 | scipy（feather） |

**RT遮罩绘制 特性**：
- 支持 CPU/GPU 处理（默认 CPU，推荐用于大图像）
- 分块处理超大数据，防止 OOM
- 支持透明度控制
- 支持多种颜色格式

**依赖说明**：
- 部分节点需要 `scipy` 或 `opencv-python`，如未安装会自动回退到基础功能

---

### ✏️ 字符串处理节点

| 节点名称 | 说明 |
|----------|------|
| RT多字符串连接 | 连接多个字符串输入，支持自定义分隔符（支持 `\n`、`\t` 等转义字符） |
| RT双字符串连接 | 双字符串连接，支持清理空白 |

**特性**：
- 所有输入都可以为空且可不连线
- 支持最多 1000 个字符串同时连接
- 分隔符支持转义字符（换行符 `\n`、制表符 `\t` 等）
- 可选择清理每个输入的首尾空白
- 可选择返回字符串列表而非连接后的字符串

---

### 🔢 数学表达式节点

| 节点名称 | 说明 |
|----------|------|
| RT数学表达式 | 执行数学表达式计算，支持动态添加输入变量 |

**特性**：
- 支持 **基本算术运算**：+、-、*、/、//、**、%
- 支持 **位运算**：&、|、^、~、<<、>>
- 支持 **逻辑运算**：and、or、not
- 支持 **比较运算**：>、<、>=、<=、==、!=
- 支持 **数学函数**：abs、round、min、max、sin、cos、tan、sqrt、log、floor、ceil 等
- 支持 **随机函数**：randomint、randomchoice
- 支持 **条件函数**：iif（类似三元运算符）
- 支持 **常量**：pi、e
- 支持 **动态输入变量**：x1、x2、x3...（最多 100 个）
- 支持 **任意类型输入**：可连接 INT、FLOAT、IMAGE、LATENT 等
- 支持 **属性访问**： IMAGE.width、IMAGE.height、LATENT.width、LATENT.height
- 支持 **跨节点引用**：NodeName.widgetName
- 支持 **UI 直接显示结果**：OUTPUT_NODE
- 完善的 **错误处理**：除以零、负数平方根等会显示详细错误信息

**使用示例**：

```
# 基础运算
x1 + x2 * 3

# 属性访问
x1.width * 0.5

# 条件判断
iif(x1 > 100, 100, x1)

# 随机数
randomint(1, 100)

# 字符串选择
randomchoice("option1", "option2", "option3")
```

---

### 🎨 颜色匹配节点

| 节点名称 | 说明 |
|----------|------|
| RT颜色匹配 | 将参考图像的颜色分布迁移到目标图像，支持多种算法和显存优化 |

**特性**：
- **多种算法支持**：reinhard_lab（推荐）、reinhard、mkl、mvgd、hm
- **GPU 加速**：reinhard_lab 算法使用 GPU 加速
- **显存优化**：支持分块处理，防止 OOM
- **设备自动降级**：GPU 失败时自动切换到 CPU
- **多线程处理**：批量处理时自动启用多线程
- **强度控制**：可调节颜色迁移强度（0-2.0）
- **进度条显示**：ComfyUI 原生进度条

**推荐配置**：
| 场景 | 推荐设置 |
|------|---------|
| 日常使用 | `reinhard_lab` + `strength=1.0` |
| 视频批量处理 | `reinhard_lab` + `batch_size=8-16` |
| 追求质量 | `mkl` |
| 显存不足 | 切换到 `cpu` 或减小 `batch_size` |

---

### 🔗 图像融合节点

| 节点名称 | 说明 | 状态 |
|----------|------|------|
| RT图像融合 | 将AI重绘结果与原始图像无缝融合，支持Poisson梯度域融合 | ⚠️ 开发中 |
| RT遮罩羽化混合 | 通过高斯模糊羽化遮罩边缘，实现自然的图像过渡效果 | ✅ 稳定 |
  
**RT遮罩羽化混合 特性**：
- **高斯模糊羽化**：通过高斯模糊对mask边缘进行羽化处理
- **参数可调**：支持调整kernel大小和sigma值控制羽化效果
- **自动尺寸适配**：自动缩放输入到统一尺寸

**使用建议**：
- 使用 **RTColorMatch + RT遮罩羽化混合** 组合获得最佳效果
- RT图像融合节点正在优化中，暂不推荐使用

### 在 ComfyUI 中搜索节点

在 ComfyUI 节点搜索框中输入 `RT_` 即可看到所有自定义节点。

### 查看帮助文档

1. 找到需要查看的节点
2. 点击节点右上角的 **绿色问号** 按钮（🟢❓）
3. 右侧会弹出该节点的详细帮助文档

### 惰性求值开关使用示例

```
工作流场景：根据条件选择不同的处理方式

1. 添加 RT惰性开关
2. 连接 condition 输入（布尔值）
3. 如果 condition=True，输出 on_true 分支
4. 如果 condition=False，输出 on_false 分支
5. 未被选中的分支不会被计算，提高效率
```

### 数学表达式使用示例

```
工作流场景：计算图像缩放比例

1. 添加 RT数学表达式
2. 调整 inputcount 为 2（x1, x2）
3. 输入表达式：x1.width / x2.width
4. 连接两个图像节点到 x1 和 x2
5. 输出结果为浮点数比例
```

### 颜色匹配使用示例

```
工作流场景：视频帧颜色统一

1. 添加 RT颜色匹配
2. 设置 method 为 reinhard_lab（GPU加速）
3. 设置 batch_size 为 8-16（根据显存调整）
4. 连接参考图像到 image_ref
5. 连接目标图像到 image_target
6. 输出为颜色匹配后的图像
```
